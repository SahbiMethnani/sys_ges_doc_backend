Documentation API REST - Système de Gestion

Authentification

Toutes les requêtes API nécessitent une authentification via token JWT.

Obtenir un token

POST /api/auth/login
Content-Type: application/json

{
    "username": "admin",
    "password": "mot_de_passe"
}

Réponse:

{
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "expiresIn": 3600
}

Endpoints Utilisateurs

GET /api/users

Récupère la liste de tous les utilisateurs

Headers: Authorization: Bearer {token}

Paramètres optionnels:

  • page: numéro de page (défaut: 1)
  • limit: nombre d'éléments par page (défaut: 10)
  • search: recherche par nom ou email

POST /api/users

Crée un nouvel utilisateur

{
    "username": "john_doe",
    "email": "john@example.com",
    "password": "SecurePass123!",
    "role": "USER"
}

PUT /api/users/{id}

Met à jour un utilisateur existant

DELETE /api/users/{id}

Supprime un utilisateur (soft delete)

Endpoints Documents

GET /api/documents

Liste tous les documents accessibles par l'utilisateur

Filtres disponibles:

  • type: PDF, WORD, EXCEL
  • createdAfter: date ISO 8601
  • createdBefore: date ISO 8601

POST /api/documents/upload

Upload un nouveau document

Content-Type: multipart/form-data

Taille maximale: 10 MB

Codes d'erreur

Code Signification
200 Succès
400 Requête invalide
401 Non authentifié
403 Accès interdit
404 Ressource non trouvée
500 Erreur serveur

Rate Limiting

L'API est limitée à 100 requêtes par minute par token.

Headers de réponse:

  • X-RateLimit-Limit: limite maximale
  • X-RateLimit-Remaining: requêtes restantes
  • X-RateLimit-Reset: timestamp de reset

Exemples avec curl

Login

curl -X POST http://localhost:8080/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"secret"}'

Lister les utilisateurs

curl -X GET http://localhost:8080/api/users \
  -H "Authorization: Bearer YOUR_TOKEN"
